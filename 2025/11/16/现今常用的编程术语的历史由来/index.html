核心转储(Core Dump)

  • 核心已转储（core dumped)，更准确的名称实际上是内存转储(memory dumped)
  • 转储文件（dump file）记录了计算机程序在特定时刻（通常是程序崩溃或异常终止时）的工作内存状态，而内存转储(memory dumped)意味着内核不仅会终止进程，还会复制整个地址空间并将其保存为磁盘文件。
    ![[Pasted image 20251116145613.png]]
  • 由于这个文件精确复制了内存空间的状态，后续可以用这个文件来分析程序的故障原因
  • 问题是，为啥要叫核心转储？为啥不叫内存快照创建或者进程状态保存？
    • 这就要追溯到早期的磁芯存储器了，那会儿计算能力有限，无法实现运行时调试器，系统崩溃后，查明原因的少数方法之一就是分析当时的内存内容，因此程序员依赖内存转储来调试问题。
    • 就像今天我们叫RAM内存一样，那会儿人们叫这东西核心（core），所以它就成了core dump而不是memory dump，至于为啥叫core嘛：
      ![[Pasted image 20251116145812.png]]
      程序崩溃的时候经常出现的错误信息： 与内存直接相关的问题是段错误（Segmentation fault），核心转储本身是个次要信息
      程序出错时使其同停止的实质是内核发送的信号，有的信号只会停掉程序，有的带一个核心转储操作
      ![[Pasted image 20251116145732.png]]
      这段的内容里，段错误（Segmentation fault） 是程序崩溃的原因，核心已转储(core dumped) 是操作系统已经保存**转储文件(dump file)**，以供后续排查问题

打印(print)

![[Pasted image 20251116150502.png]]
老老年的时候抛弃了打孔卡带，开始用电传打字机，这边打的字符会显示在纸上，也会发送给计算机，计算机也会把输出返回回来打印在纸上
ASCII码的前32位不可print，因为他们是控制字符，电传打字机打了这些字符就会触发电传打字机的机械动作，比如移回行首，换行叫要把纸张转筒转一下，移动到下一行
![[Pasted image 20251116150659.png]]
顺带一提，Unix系统是肯-汤普森和丹尼斯-里奇用电传打字机开发的
![[Pasted image 20251116150934.png]]
再后来电传机被哑终端取代，实际上就是换成了一张电子的纸，这个时候来到了辐射4(fallout4)时代
![[Pasted image 20251116151121.png]]
顺带一提，光标最早是计算尺上滑动标记的名称，用于追踪刻度移动轨迹
![[Pasted image 20251116151225.png]]
再到今天的终端模拟器
![[Pasted image 20251116151415.png]]
那么为啥终端也被称为控制台(Console) 呢？
因为上古时代，连电传打字机都没出现的时代，要用这种带开关和指示灯的巨大面板来控制计算机，1960年代电传打字机出现之后迅速取代了这个东西，不过这个名字依旧留了下来，到1970年代哑终端问世的时候它还是会被人习惯性的称之为Console
