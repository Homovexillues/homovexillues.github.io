PostgreSQL数据库

介绍

数据页

![[Pasted image 20251116162917.png]]
![[Pasted image 20251116162936.png]]
pg的数据表叫堆表文件，是磁盘上一个数字命名的没有后缀名的文件这个数字代表这张数据表在数据库里的ID，又叫relfileNode,数据表越大，堆表文件也就越大，而当堆表文件超过1GB时，会自动分割为多个文件
![[Pasted image 20251116172222.png]]
堆表文件里会将数据拆分成一个一个数据页来存储，每页8KB，这样只需要读取磁盘里的页就可以，比读写整个表要快很多
![[Pasted image 20251116172914.png]]

安装

Linux

linux下如果apt能找到直接安装

如果找不到加个源再装,官方文档地址:PostgreSQL: Linux downloads (Ubuntu)

装完了之后会自动注册成systemctl服务

且会自动配置开机自启动

配置

配置文件

修改postgresql.conf

Linux下位置在：/etc/postgresql/15/main/postgresql.conf

修改同文件夹下pg_hba.conf

常用元命令
命令 说明 示例输出片段
\l 列出所有数据库 `mydb \ myuser \ UTF8`
\c dbname 切换数据库 You are now connected to database "mydb"
\dt 列出当前数据库的所有表 `public \ users \ table`
\d+ tablename 查看表结构详情 列名、类型、约束等信息
\q 退出 psql -

基础数据库

OID 数据库名 作用
1 template1 真正的“模板库”。以后你执行 CREATE DATABASE 时，PG 会直接把 template1 复制一份成新库。
4 template0 只读备胎模板，内容与 template1 几乎一样，但编码/区域设置不可改，用于“干净”重建或需要不同编码时。
5 postgres 默认给你连接的管理库，啥也没有，纯粹方便管理员连上来再建别的库。

PostgreSQL 自己必须存在的三个系统数据库，在执行 initdb 时就自动生成，永远不能删掉
