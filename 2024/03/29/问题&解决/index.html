  • 方法和函数的区别是什么？

    • 一般在面向对象的语言中，这个被称为方法，面向过程语言中被称为函数，没啥太大区别，都是接收输入的数据并返回数据
    • 仅有的一点区别是：方法与其被调用的对象实例相关联，方法仅限于在其所包含的类中操作数据
    • 在C++中可以创建与对象关联的函数，被称为成员函数，在其它语言中就是方法

  • C#中[STATHread]是什么意思？

    • [STAThread]是一个特性，用于指示应用程序的默认线程是单线程单元(STA),多用在程序的入口方法上
    • STA即Single Thread Apartment，是一种线程模型，在这个模型中，每个线程都有自己的单独的消息队列，用于处理Windows消息。这种模型对于需要处理用户界面的线程非常有用，因为用户界面通常需要在单个线程上进行操作以避免并发问题
    • [STATHread]特性在使用COM Interop(托管代码与非托管代码之间的协同过程)的应用程序中非常重要。如果应用程序实际调用了COM组件，则仅为COM interop初始化该线程，如果没有使用COM interop,则不初始化该线程。

  • #region…#endregion是什么意思？

    • 在C#中，#region和#endregion是预处理指令，用于在VisualStudio中标记一块可以折叠和展开的代码区域，可以在#region后添加任意描述文字，以便记住此区域的用途，不会影响代码执行，只是提升可读性

  • ///<summary>...</summary>写法是什么意思？

    • 这是一种特殊的注释，被称为XML文档注释，这种注释可以被VisualStudio的IntelliSense（智能感知）和其他工具用来生成代码的文档
    • 可以在属性、方法等前面输入///来自动生成这种注释

  • var是什么意思

    • var是一个关键字，用于声明变量，var关键字告诉编译器，希望它根据变量的初始化表达式来推断变量类型。这被称为类型推断。
    • 如var total =0;编译器会推断出total是一个int类型的变量，因为它被初始化为一个整数。
    • var声明的变量必须在声明的时候进行初始化
    • var只能用于局部变量的声明，不能在类的级别使用var来声明字段。

    • 委托可以封装一个方法的签名，可以视作对方法的引用，因此，定义一个委托类型的属性，这个属性可以存储一个方法的引用，然后可以通过这个属性来调用这个方法。
    • 委托对其封装的方法没有直接的影响。可以将方法作为参数传递给其他方法，或者将多个方法组合在一起，此外，使用位图哦可以实现事件和回调函数。
    • 委托的参数是指通过委托调用方法时传递的参数。

  • internal static Action<string, object> Write { get; set; }是什么意思

    • internal访问修饰符，只能在同一程序集中被访问
    • static修饰符，表示这个属性属于类本身，而不是类的实例。可以通过类名来访问这个属性而不需要创建类的实例。
    • Action<string ,object>这是属性的类型。Action代表无返回值的方法的委托。这个Action委托接受两个参数，一个是string类型，一个是object类型。
    • Write属性名称
    • {get;set;}属性的访问器。
    • internal static Action<string, object> Write { get; set; }定义了一个名为Write的属性，它的类型是Action<string,object>可以在同一个程序集中被访问，且属于类本身。
    • 这个Write属性可以存储一个方法的引用，这个方法必须接受一个string参数和一个object参数，并且没有返回值。可以通过Write属性来调用这个方法。

  • 什么是语法糖？

    • 语法糖是一种术语，指的是设计来使编程更容易或更可读的语言特性。这些特性并没有增加语言的功能，但是可以使代码更简洁，更易于阅读和编写。
    • 如，在C#中，属性(properties)就是一种语法糖。可以直接使用obj.Property来获取或设置值，而不需要调用obj.GetProperty或obj.SetProperty(value)。这使得代码更简洁，更像自然语言。
    • 虽然语法糖可以提高代码的可读性和编写效率，但是过度的使用可能会使代码变得难以理解和维护。

  • 什么是并发字典（ConcurrentDictionary）？

    • 并发字典是.NET Framework中的一个类,System.Collections.Concurrent命名空间的一部分。
    • 并发字典是线程安全的字典，这意味着多个线程可以同时从字典中添加、删除和更新键值对，而不会引发任何线程冲突或数据不一致的问题。
    • 并发字典的主要优点是它允许多个线程同时读取和写入数据，而不需要使用锁或其他同步机制。这使得并发字典在需要高并发访问的多线程应用程序中非常有用。

  • .dll文件和python库之间有什么关系？

    • .dll文件是动态链接库（Dynamic Link Library）的缩写，它包含了可以被多个程序共享的代码和数据。这些库通常是用C或C++编写的。
    • 如numpy库的源代码中有大量的C代码，这些代码在安装numpy库的时候会被编译为动态链接库。然后在python中通过import numpy调用库的时候，python会加载动态链接库，并将其中国的函数和数据结构结构映射到python的命名空间中。
    • 如调用numpy.arrary()的时候，实际上是在调用一个C语言实现的函数。这个函数会在C语言的环境中执行，然后将结果返回给python。

  • 类类型的属性和实例有啥区别？

    • 主要区别在于访问方式和生命周期
    • 类类型的属性:类属性可以有默认值，也可以在构造函数中初始化。它们的声明周期与类实例的声明周期相同。
    • 实例:实例是类的具体化。实例的声明舟曲从创建它的那一刻开始，直到没有任何引用指向它并被垃圾收集器回收为止。

  • 什么是锁语句？

  • 在判断中，is not null和!=null有什么区别吗？

    • 在大多数编程语言中没啥区别

    • 在SQL中，这两者有重要区别。NULL是一种特殊的值，表示未知或缺失的数据。因此，NULL与任何值，包括NULL自己的比较结果都是未知的，即NULL=NULL和NULL!=NULL都不会返回true

    • 因此当想要检查一个字段是否为NULL的时候，应该用IS NULL或IS NOT NULL。

    • 如WHERE column IS NOT NULL会选取那些column字段不为NULL的行。

  • .yaml .json .xml文件格式各自用于做什么？有什么使用上要注意的事？

    • JSON（JavaScript Object Notation）：json简介易读，适用于API的请求和响应。它是一种轻量级的数据交换格式，易于人阅读和编写，同时也易于机器解析和生成。JSON采用完全独立于语言的文本格式，使得JSON成为理想的数据交换语言。
    • XML(eXtensible Markup Language): XML具有跨平台和跨语言的优点，适用于Web服务和RSS源。它是一种标记语言，可以用来编码文档和数据。XML的设计目标是强调通用性和简单性。尽管XML的使用在web上的数据序列化的用途已经减少，但它仍然被广泛用于存储和传输数据。
    • YAML(YAML Ain’t Markup Language)：YAML适合用于需要人类可读的数据表示场景，如配置文件和日志文件。YAML是一种人类友好的数据序列化标注，它允许在不同语言之间交换数据。YAML是对JSON的超集，因此任何JSON文件都应该能够用YAML解析器解析。
    • XML用标记来定义结构数据，就像html那样
    • YAML使用缩进来定义结构化数据。就像python
    • JSON使用的是冒号和大括号来定义结构化数据
    • 数据传输的时候，XML和JSON很常见，尤其是JSON，在API的开发中变得非常普及
    • 文件大小上来说，同样的数据集。YAML文件最小，JSON次之，XML最大
    • 学习难度上来说，YAML最难，JSON和XML相对简单

  • Task.Delay()和Task.Wait()和Task.Sleep()的区别是什么？

    • Task.Delay():一个异步的延迟方法，返回一个任务，在指定的时间间隔后该任务会完成。这个方法是非阻塞的。
    • Task.Wait()：一个同步的方法，它会阻塞当前的线程，直到任务完成。
    • Task.Sleep()：阻塞当前线程，使其在指定时间间隔进入睡眠。

  • C# 中nameof是如何使用的？

    • nameof是一个编译器关键字，用于获取变量、类型、方法、属性、事件、索引器和命名空间的名称作为字符串常量。nameof提供了一种方便的方式来引用这些 实体的名称，而不需要硬编码字符串，当实体名称改变时，相关的字符串也会自动更新，减少出错的可能性。

  • .ico文件和.resx文件是啥？.config文件一般是什么格式的？

    • .ico:图标文件
    • .resx:资源文件格式，用于存储本地化字符串、图像和其他资源。通常用于.NET Framework应用程序，以便用于管理和本地化应用程序的资源。
    • .config:在.NET应用程序中，包含应用程序的设置，如数据库连接字符串，应用该程序设置等。XML格式，允许开发者以结构化的方式定义配置选项。
