C#发展史

.NET 的发展历史可以分为多个阶段，涵盖了 .NET Framework、.NET Core 和 ASP.NET Core 的演变。以下是一个简要的概述：

1. .NET Framework （2002年发布）

  • 概述：这是微软在2002年首次推出的开发平台，旨在简化 Windows 应用开发。它包括了一个庞大的类库（BCL - Base Class Library）和 Common Language Runtime (CLR)，支持多种编程语言（如 C# 和 VB.NET）。
  • 主要特点：
    • 主要用于开发桌面应用（Windows Forms、WPF）、Web应用（ASP.NET Web Forms、ASP.NET MVC）、以及企业级应用。
    • 由于紧密集成 Windows 操作系统，跨平台支持较差。
  • 发展：
    • 通过多个版本的迭代添加了如 WCF（Windows Communication Foundation）、WPF（Windows Presentation Foundation）等技术。
    • .NET Framework 的最终版本是 4.8，在此之后，微软将更多的精力投入到跨平台的 .NET Core 上。

2. .NET Core （2016年发布）

  • 概述：为了适应现代应用需求，微软在2016年推出了 .NET Core，作为一个轻量、跨平台的框架。它主要面向云计算、微服务和跨平台应用。
  • 主要特点：
    • 支持跨平台开发（Windows、Linux、macOS）。
    • 模块化设计，开发者可以只安装和使用所需的组件。
    • 性能改进显著，特别是在 Web 和云端应用中。
    • 开源，微软通过 GitHub 维护 .NET Core。
  • 发展：
    • 最初版本是 .NET Core 1.0，功能较少，主要支持控制台应用和 ASP.NET Core。
    • .NET Core 2.x 引入了更多的 API，使其更接近 .NET Framework 的功能。
    • .NET Core 3.0 开始支持 Windows Forms 和 WPF 等桌面应用，进一步加强其跨平台能力和性能优化。

3. ASP.NET 和 ASP.NET Core

  • **ASP.NET (2002年发布)**：

    • 这是基于 .NET Framework 的 Web 开发框架，最早的版本是基于 Web Forms，支持服务器端控件和事件驱动的开发模式。
    • 后来推出了 ASP.NET MVC，提供了基于 MVC 模式的开发方式，增加了更多的灵活性和可测试性。

  • **ASP.NET Core (2016年发布)**：

    • ASP.NET Core 是重新设计的 Web 开发框架，最初是为 .NET Core 打造的，后来也兼容 .NET Framework。其最大的优点是跨平台和高性能。
    • 与传统的 ASP.NET 不同，ASP.NET Core 是完全模块化的，开发者可以根据需要选择不同的中间件来处理请求和响应。
    • 逐步淘汰了 Web Forms，主推基于 Razor 的页面或 MVC 模式的开发。

4. 统一的 .NET 平台：**.NET 5 和之后** （2020年发布）

  • 概述：.NET 5 在2020年发布，标志着微软正式将 .NET Framework 和 .NET Core 合并为一个统一的开发平台，称为 “.NET”。从这一版本开始，微软每年发布一个新的主要版本。
  • 特点：
    • 继续跨平台支持，提升性能。
    • 统一了应用模型，开发者不再需要区分 .NET Core 和 .NET Framework。
    • 提供了对更多平台和框架的支持，如 WebAssembly 和 MAUI（用于跨平台的移动和桌面应用开发）。

5. .NET 6 和 .NET 7

  • **.NET 6 (2021年发布)**：

    • LTS（长期支持版），进一步提升了开发效率和跨平台能力。
    • 引入了 MAUI 框架，使开发者能够用一个代码库编写跨平台的桌面和移动应用。

  • **.NET 7 (2022年发布)**：

    • 提供更多的性能优化和新特性，进一步整合各种技术栈，如 Blazor（前端框架）、云原生应用支持等。

关系和演变：

  • .NET Framework：最早期的微软开发平台，专注于 Windows 应用。
  • .NET Core：后期发展出的一种轻量、跨平台框架，逐渐成为主流。
  • ASP.NET Core：用于构建 Web 应用的现代框架，基于 .NET Core 构建。
  • **.NET 5+**：新的统一平台，取代了 .NET Framework 和 .NET Core，成为跨平台开发的未来方向。

总结：

  • .NET Framework 专注于 Windows，但随着时代发展，跨平台需求增加，微软转向了 .NET Core。
  • .NET Core 强调性能、模块化和跨平台能力。
  • .NET 5 及以后 通过统一平台，将 .NET Core 的现代特性与 .NET Framework 的广泛功能结合，继续适应现代开发需求。
