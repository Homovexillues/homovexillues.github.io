计算机网络零碎知识点

私有 IP 地址范围有三段

根据 RFC 1918，私有 IP 地址范围有三段，每段都是专门保留给局域网使用的，不会在公网中出现：

  1. 10.0.0.0 - 10.255.255.255（10.0.0.0/8）：适用于大规模的私有网络（例如大型企业内部网络）。
  2. 172.16.0.0 - 172.31.255.255（172.16.0.0/12）：适用于中等规模的私有网络。
  3. 192.168.0.0 - 192.168.255.255（192.168.0.0/16）：适用于小型私有网络，也是最常见的家用和小型办公室网络范围。

子网掩码的原理

子网掩码的作用就是分割Ip中网络和设备的部分

在 IPv4 地址中（例如 172.21.10.67），一个 IP 地址由 32 位组成，通常分成 4 组，每组 8 位（1 个字节），用十进制表示成 x.x.x.x 的形式。

子网掩码也是一个 32 位的数字，它用来遮蔽 IP 地址的前几位以表示网络部分。子网掩码中的每个 1 表示网络部分，每个 0 表示主机部分。例如，255.255.255.0 的二进制形式是 11111111.11111111.11111111.00000000，表示前 24 位（即前三组 8 位）属于网络部分，最后 8 位属于主机部分。

对于 IP 地址 172.21.10.67 和子网掩码 255.255.255.0，转换为二进制为：

  • IP 地址：10101100.00010101.00001010.01000011
  • 子网掩码：11111111.11111111.11111111.00000000

子网掩码的标准格式

在二进制下，子网掩码的1或0必须连续

标准的子网掩码格式只允许使用以下形式：

  • 255.0.0.0 (即 /8)
  • 255.255.0.0 (即 /16)
  • 255.255.255.0 (即 /24)
  • 以及其他以 1 位连续、0 位连续的格式，比如 255.255.252.0 (即 /22)

如何计算一个子网的 IP 地址数量

$$
\text{可用 IP 地址数} = 2^{(32 - \text{子网掩码中的1的数量})} - 2
$$

  • 255.255.255.0 (/24)：32 - 24 = 8，所以可用 IP 数是 $ 2^8 - 2 = 256 - 2 = 254$
  • 255.255.252.0 (/22)：32 - 22 = 10，所以可用 IP 数是 $2^{10} - 2 = 1024 - 2 = 1022 $

ARP(Address Resolution Protocol,地址解析协议)是一种网络协议

ARP地址解析协议用于将IP地址解析为物理地址(MAC地址)。

在局域网中，设备通过 IP 地址识别对方，但实际的数据传输是通过 MAC 地址完成的。ARP 的主要作用是找到目标 IP 地址所对应的 MAC 地址，以便正确地传输数据。

ARP广播就是设备通过IP找对应MAC的过程

当设备 A 想要与设备 B 通信时，设备 A 需要知道设备 B 的 MAC 地址，但它可能只知道设备 B 的 IP 地址。

设备 A 会发送一个 ARP 广播请求，内容大致为“谁的 IP 地址是 X.X.X.X？请告诉我你的 MAC 地址。”

这个广播会被发送到同一局域网的所有设备（因为是广播），只有 IP 地址匹配的设备（即设备 B）会回应这个请求。

设备 B 收到请求后，会发送一个 ARP 响应，告知设备 A 其 MAC 地址。

设备 A 获取到设备 B 的 MAC 地址后，就可以通过该 MAC 地址将数据发送给设备 B。

ARP缓存就是IP与MAC的对应表

ARP 缓存是设备维护的一个表，用于存储 IP 地址和 MAC 地址之间的映射关系。

当设备 A 获取了设备 B 的 MAC 地址后，它会将该 IP 地址和 MAC 地址对暂时存储在 ARP 缓存中。

这样，当设备 A 下次需要和设备 B 通信时，可以直接从 ARP 缓存中查找，不需要再次发送 ARP 广播请求。

ARP 缓存中的记录通常有一定的生存时间（TTL），过期后会自动清除。如果设备 A 再次需要与设备 B 通信，则需要重新发送 ARP 请求。
